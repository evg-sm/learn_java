# Fail-Fast и Fail-Safe итераторы в Java

## Fail-Fast итераторы

- **Поведение**: Выбрасывают `ConcurrentModificationException` при модификации коллекции во время итерации (кроме операций самого итератора).
- **Как работает**: Использует счетчик модификаций (`modCount`) для обнаружения изменений.
- **Примеры**: `ArrayList`, `HashMap`, `HashSet`.
- **Пример кода**:
```java
List<String> list = new ArrayList<>(List.of("A", "B"));
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (s.equals("A")) {
        list.remove("A"); // Выбросит ConcurrentModificationException
    }
}
```

## Fail-Safe итераторы

- **Поведение**: Работают со снимком данных, поэтому модификации оригинальной коллекции не влияют на итерацию. Исключения не возникают.
- **Как работает**: Создает копию коллекции в начале итерации.
- **Примеры**: `CopyOnWriteArrayList`, `ConcurrentHashMap`.
- **Пример кода**:
```java
List<String> list = new CopyOnWriteArrayList<>(List.of("A", "B"));
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (s.equals("A")) {
        list.remove("A"); // Исключение не возникает
    }
}
```

## Основные различия
| Характеристика | Fail-Fast | Fail-Safe |
|----------------|-----------|-----------|
| **Исключение** | Выбрасывает `ConcurrentModificationException` | Нет исключения |
| **Согласованность данных** | Отражает текущее состояние, но падает при модификациях | Работает со снимком; может не видеть последние изменения |
| **Сценарий использования** | Однопоточные сценарии | Многопоточные среды |
| **Производительность** | Низкие накладные расходы при итерации | Высокие накладные расходы при модификациях (копирование) |

## Когда использовать
- **Fail-fast**: Подходит, когда нужно немедленное обнаружение параллельных модификаций (например, отладка).
- **Fail-safe**: Идеален для многопоточного доступа, где модификации во время итерации ожидаются (например, системы с высокой нагрузкой).
