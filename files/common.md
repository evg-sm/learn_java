### HTTP header: keep-alive - зачем нужен?  
**_Connection: keep-alive_** - Если в запросе отправлено значение keep-alive, 
то соединение остаётся и не завершается, позволяя выполнять последующие запросы на тот же сервер.  

### Каким образом передаются переменные в методы, по значению или по ссылке?
В Java параметры всегда передаются **_только по значению_**, 
что определяется как **_«скопировать значение и передать копию»_**. 
С примитивами это будет копия содержимого. Со ссылками - тоже копия содержимого, т.е. копия ссылки. 
При этом внутренние члены ссылочных типов через такую копию изменить возможно, а вот саму ссылку, 
указывающую на экземпляр - нет.

###  Императивное vs декларативное программирование  

**_Java_** - изначально императивный язык с элементами декларативного.

**_Императивный стиль_** - Это такой стиль программирования, при котором описывается, **_как добиться желаемого результата_**.
Например, некий алгоритм с помощью которого можно получить результат.  
**_Императивный стиль отвечает на вопрос - Как делать?_**

**_Декларативный стиль_** - Эта такой стиль программирования, при котором описывается, **_какой именно результат вам нужен_**.  
**_Декларативный стиль отвечает на вопрос - Что делать?_**

**_На примере фильтрации коллекций:_**

`List<Integer> integers = List.of(1, 2, 3,4, 8, 10, 22 ,434, 67, 4, 5, 6);`  

_Фильтрация в императивном стиле:_  
`for (int i = 0; i < integers.size(); i++) { if(integers.get(i) >= 10) { result.add(integers.get(i)); } }`  

_Фильтрация в декларативном стиле:_  
`integers.stream().filter(integer -> integer >= 10).toList();`

**_Т.е. фильтрация циклом - Императивный стиль, через stream api - Декларативный стиль._**  

Хорошим примером декларативного стиля программирования в Java является генерация rest клиентов библиотекой openFeign,
или генерация классов из xsd схемы посредством специальных плагинов.

**_Примеры декларативных языков программирования:_**
- SQL
- HTML

**_Императивные языки:_**
- C,
- C++
- Java  

### Что такое HEAD в GIT  
HEAD в GIT - это указатель на текущую ветку, которая, в свою очередь, является указателем на последний коммит, 
сделанный в этой ветке. Это значит, что HEAD будет родителем следующего созданного коммита. 
**_Как правило, самое простое считать HEAD снимком вашего последнего коммита._**
